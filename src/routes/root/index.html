Monk API is a multi-tenant backend runtime for model-first systems. This page is intentionally compact: it tells LLMs where to look, how to orient, and which routes are canonical before they start calling the API.

Humans can read this page, but the copy and structure are optimized for agent navigation and endpoint discovery.

How to start

  • Read /llms.txt first.
  • Use /docs for API discovery.
  • Prefer documented route shapes over guessing.
  • Inspect route docs before writing requests.

What this API exposes

  • /api/data for CRUD and traversal
  • /api/describe for schema definition
  • /api/find and /api/aggregate for queries
  • /api/bulk, /api/cron, /fs

Operational facts

  • Default port: 9001
  • Auth: JWT and API keys
  • Tenants are isolated
  • Docs live beside the routes
